# SoriMemo 작업 로그 - 2026-06-21

## 작업 범위

- 회원가입 완료 후 바로 서비스로 들어가지 않고, 회원가입 과정에서 동의 절차를 이어서 받은 뒤 로그인 화면으로 이동하도록 수정했습니다.
- 로그인 시 기존 동의 기록을 확인하는 `GET /api/consent/latest` API를 추가했습니다.
- 기존 동의가 있는 사용자는 로그인 후 동의 절차를 반복하지 않고 서비스 시작 화면으로 이동하도록 프론트 흐름을 수정했습니다.
- 서비스 상단의 `홈`, 뒤로가기, 첫 화면 `회원가입` 버튼 글씨 크기를 다른 화면 제목과 비슷하게 키웠습니다.
- `지난 검증 이력` 화면 제목을 다른 주요 서비스 화면 제목과 같은 큰 제목 크기로 맞췄습니다.
- 검증 이력 화면의 최근 요약과 목록 숫자 기준을 `참고값`으로 통일했습니다.
- 검증 이력 화면에서 혼란을 만들던 `결과 신뢰도 75%` 표시는 제거하고, AI 참고값과 위험 신호 해석 문구를 표시하도록 수정했습니다.
- 학습용 장기보관 동의 데이터 분석 시 `audio_files.status = training_retained`가 DB 체크 제약조건에 막히는 문제를 해결하기 위한 마이그레이션을 추가했습니다.

## 운영 반영

- 프론트엔드 빌드를 여러 차례 수행했고, 변경된 `dist` 산출물을 `/var/www/sorimemo/`에 배포했습니다.
- 백엔드 변경 사항은 `sorimemo-backend` 재시작 후 반영됩니다.
- DB 마이그레이션 파일은 PostgreSQL 사용자가 읽을 수 있도록 `/tmp`로 복사한 뒤 실행해야 합니다.
